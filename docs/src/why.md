## References

- [The Rust Programming Language Book](https://doc.rust-lang.org/stable/book/): the main introductory book about Rust, usually referred to as "the book" by rustaceans
- [Rust by Example](https://doc.rust-lang.org/rust-by-example/): a collection of runnable examples that illustrate various Rust concepts and standard libraries. Usually consumed alongside "the book", gives a more interactive experience to learning Rust basics
- [The Rust Reference](https://doc.rust-lang.org/reference/introduction.html): a book that serves as a reference to what is available in stable Rust.
- [The Rust Standard Library Documentation](https://doc.rust-lang.org/std/index.html)
- [crates.io](https://crates.io/): The Rust community’s crate registry
- [The Cargo book](https://doc.rust-lang.org/cargo/index.html): `cargo` is the Rust package manager and build system
- [The rustdoc Book](https://doc.rust-lang.org/rustdoc/what-is-rustdoc.html): The standard Rust distribution ships with a tool called `rustdoc`. Its job is to generate documentation for Rust projects.
- [Rust API Guidelines](https://rust-lang.github.io/api-guidelines/): This is a set of recommendations on how to design and present APIs for the Rust programming language. They are authored largely by the Rust library team, based on experiences building the Rust standard library and other crates in the Rust ecosystem
- [Command Line Applications in Rust](https://rust-cli.github.io/book/index.html): Rust is a statically compiled, fast language with great tooling and a rapidly growing ecosystem. That makes it a great fit for writing command line applications: They should be small, portable, and quick to run. Command line applications are also a great way to get started with learning Rust; or to introduce Rust to your team!
